# WwwLove — сервер (бэкенд)

Реализует то, чего не хватало прототипу: **настоящую привязку двух телефонов** и обмен ответами через интернет. На Node.js + встроенный SQLite (без внешних баз и без сервисов Google — подходит для RuStore-аудитории).

## Что умеет
- Регистрация устройства (анонимно, по device_id)
- Создание кода пары и привязка второго телефона по коду
- Хранение даты начала отношений и статуса премиума (на паре, а не на устройстве)
- Приём ответов и выдача «Книги пары» с обоими ответами и пометкой совпадения

Логика проверена end-to-end: устройство A создаёт код → B привязывается → оба отвечают → книга показывает оба ответа и определяет совпадение.

## Запуск локально
```bash
npm install
npm start
# сервер на http://localhost:3001
```
SQLite-файл `wwwlove.db` создаётся автоматически рядом с проектом.

## API (кратко)
- `POST /api/register` `{deviceId}` → `{userId, pair}`
- `POST /api/pair/create` `{deviceId}` → `{code, pair}`
- `POST /api/pair/join` `{deviceId, code}` → `{pair}`
- `GET  /api/pair/status?deviceId=...` → `{pair}` (опрос: привязался ли партнёр)
- `POST /api/pair/start-date` `{deviceId, startDate}`
- `POST /api/pair/premium` `{deviceId}`
- `POST /api/answers` `{deviceId, catId, qText, answer, isChoice}`
- `GET  /api/answers?deviceId=...` → `{entries:[{q,isChoice,mine,theirs,matched}]}`

## Как подключить приложение
В приложении (`wwwlove-app`) есть готовый клиент `src/api.js`.
1. Откройте `src/api.js`, замените `API_BASE` на адрес вашего сервера.
2. В `App.jsx` вызовы-заглушки замените на методы `api.*` (комментарий внизу `api.js` показывает где что).

## Развёртывание (чтобы сервер жил в интернете)
Без сервисов Google, на российских площадках:
- VPS у российского провайдера (Selectel, Timeweb, REG.RU, Yandex Cloud Compute и т.п.).
- Поставить Node.js 22+, скопировать проект, `npm install`, запустить через `pm2` или systemd, чтобы работал постоянно.
- Поставить домен и HTTPS (например, через nginx + бесплатный сертификат). HTTPS обязателен — приложение шлёт личные данные.

## Что улучшить перед боевым запуском
- Перейти с SQLite на PostgreSQL при росте нагрузки (схема совместима).
- Добавить нормальную аутентификацию (сейчас доверяем device_id) — например, вход по номеру телефона, чтобы пара не терялась при смене устройства.
- Серверную проверку покупок (RuStore Billing) перед выставлением premium.
- Пуш-уведомления через RuStore Push SDK (отдельный серверный модуль отправки).
- Ограничение частоты запросов (rate limiting) и базовую защиту от перебора кодов.
