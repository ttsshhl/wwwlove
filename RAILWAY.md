# Как поставить сервер WwwLove на Railway (просто, без SSH)

Railway запускает наш сервер и сам выдаёт HTTPS-адрес. Никаких nginx и сертификатов
вручную. Данные (пары, ответы) хранятся на постоянном диске и не теряются.

## Что нужно
- Аккаунт на railway.app (вход через GitHub — удобнее всего).
- Папка wwwlove-server (она уже готова к Railway).

================================================================
ВАРИАНТ 1. Через GitHub (рекомендую — потом легко обновлять)
================================================================

## Шаг 1. Залить код на GitHub
1. Создай репозиторий на github.com (например wwwlove-server), приватный.
2. Загрузи туда содержимое папки wwwlove-server
   (можно через кнопку "Add file → Upload files" прямо в браузере).

## Шаг 2. Создать проект на Railway
1. railway.app → New Project → Deploy from GitHub repo.
2. Выбери свой репозиторий wwwlove-server.
3. Railway сам поймёт, что это Node.js, и запустит сборку.

## Шаг 3. Добавить постоянный диск (чтобы данные не терялись)
1. В проекте открой сервис → вкладка Variables → New Variable:
   - Имя: DB_DIR
   - Значение: /data
2. Вкладка Settings → Volumes → Add Volume:
   - Mount path: /data
   (это папка, где будет лежать база wwwlove.db — она переживает передеплои)

## Шаг 4. Получить адрес
1. Settings → Networking → Generate Domain.
2. Получишь адрес вида https://wwwlove-server-production.up.railway.app
3. Проверь: открой этот адрес в браузере — должно показать
   {"ok":true,"service":"wwwlove"}

## Шаг 5. Прописать адрес в приложении
В app/src/main/java/ru/wwwlove/app/data/Api.kt:
```
const val BASE_URL = "https://ВАШ-АДРЕС.up.railway.app"
```
(без слэша в конце). Пересобери приложение — готово, HTTPS уже работает.

================================================================
ВАРИАНТ 2. Через Railway CLI (без GitHub)
================================================================
1. Установи Node.js на свой компьютер, затем:
   npm i -g @railway/cli
2. railway login
3. В папке wwwlove-server:
   railway init
   railway up
4. Добавь Volume и переменную DB_DIR=/data (Шаг 3 выше) через сайт railway.app.
5. railway domain — получить адрес. Дальше как Шаг 5.

================================================================
ВАЖНО
================================================================
- Порт менять не нужно: Railway сам задаёт переменную PORT, сервер её читает.
- Версия Node: в package.json указано >=22.5.0 (нужно для встроенной базы) —
  Railway поставит подходящую сам.
- Диск /data обязателен. Без него база ляжет во временную папку и обнулится
  при следующем передеплое — пары и ответы пропадут.
- Если позже вырастет нагрузка — можно перейти на отдельную базу PostgreSQL
  (Railway даёт в один клик), но для старта постоянного диска достаточно.

================================================================
ОБНОВЛЕНИЕ СЕРВЕРА
================================================================
- Через GitHub: просто залей новые файлы в репозиторий — Railway передеплоит сам.
- Через CLI: railway up из папки сервера.

================================================================
ЧАСТЫЕ ВОПРОСЫ
================================================================
- "Сервер падает при старте" → проверь логи (вкладка Deployments → View Logs).
  Чаще всего: не добавлен Volume /data или переменная DB_DIR.
- "Приложение не связывается" → проверь BASE_URL в Api.kt (https, без / в конце),
  пересобрал ли приложение.
- "Адрес открывается, но {"ok":true} нет" → сервис ещё собирается, подожди минуту.
